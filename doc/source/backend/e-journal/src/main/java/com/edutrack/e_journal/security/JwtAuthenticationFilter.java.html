Summary
This component intercepts incoming HTTP requests to validate JSON Web Tokens for the EduTrack school e-journal system. It extracts the Bearer token from the Authorization header and verifies its signature and expiration using JwtUtils. Upon successful validation, it retrieves the user's email and loads their details from the database to establish a Spring Security context. The filter logs detailed audit information including timestamps, IP addresses, and user roles for both successful and failed authentication attempts. If the token is invalid or the user is not found, the request proceeds without authentication but with a warning log. This mechanism ensures that only authenticated users with valid credentials can access protected resources in the application.
