Summary
This filter provides a development convenience by automatically granting full administrative privileges to requests originating from localhost when the allow-localhost flag is enabled. It checks if the request comes from a loopback address and lacks an existing authentication token. If both conditions are met, it creates a new authentication object with all system roles, effectively bypassing standard security checks. This mechanism simplifies local testing by allowing developers to access protected endpoints without valid JWTs. The feature is explicitly disabled in staging and production environments to maintain security integrity. It is registered directly in the security configuration rather than as a standard Spring component.
