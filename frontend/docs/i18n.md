# Internationalization (i18n)

EduTrack supports **English** and **Bulgarian** using `i18next` + `react-i18next`.

## Setup

**File:** `src/i18n/index.js`

```js
i18n
  .use(initReactI18next)
  .init({
    resources: { en: { translation: en }, bg: { translation: bg } },
    lng: localStorage.getItem('language') || 'bg',   // default Bulgarian
    fallbackLng: 'en',
    interpolation: { escapeValue: false },
  });
```

Language preference is saved to `localStorage` under the key `'language'`.

## Switching language

The `Sidebar` renders two `ToggleButton` components (EN / BG):

```js
i18n.changeLanguage('bg');
localStorage.setItem('language', 'bg');
```

## Usage in components

```js
import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t } = useTranslation();
  return <Typography>{t('nav.home')}</Typography>;
}
```

## Translation file structure

Both `en.json` and `bg.json` share the same key hierarchy:

```
nav.*           Navigation menu labels
login.*         Login page
common.*        Shared labels (Cancel, Save, Delete, etc.)
roles.*         Role display names (ADMIN → "Admin")
welcome.*       Welcome banner text
users.*         User management (ManageUsers)
schools.*       School management (ViewSchools, HeadmasterSchool)
subjects.*      Subject management
schedule.*      Schedule editor
lectureTypes.*  STANDARD / SIP / EXTRACURRICULAR labels
selectClass.*   Class selection page text
selectSchool.*  School selection page text
selectTeacher.* Teacher selection page text
entryTypes.*    LECTURE / BREAK / SPECIAL_EVENT labels
schoolTypes.*   School type labels (ELEMENTARY, etc.)
grades.*        Grade pages
absences.*      Absence pages
complaints.*    Complaint pages
stats.*         Statistics pages
termConfig.*    Term date configuration
students.*      Student enrollment
teachers.*      Teacher management
parents.*       Parent management
profile.*       Profile page
```

## Key naming conventions

- Namespace: flat (single `translation` namespace)
- Separator: `.` (dot)
- Case: camelCase for key names
- Interpolation: `{{variable}}` syntax, e.g.:
  ```
  "expelConfirm": "Are you sure you want to expel {{name}}?"
  ```
  Used as:
  ```js
  t('students.expelConfirm', { name: studentName })
  ```

## Adding a new translation key

1. Add the key to `src/i18n/locales/en.json` under the appropriate section.
2. Add the Bulgarian translation to `src/i18n/locales/bg.json` under the same key.
3. Use `t('section.key')` in the component.

If a key is missing in the active language, i18next falls back to English automatically.
